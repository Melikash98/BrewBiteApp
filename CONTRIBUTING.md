# Contributing to BrewBite

Thank you for your interest in contributing to BrewBite!

We welcome bug fixes, documentation improvements, UI/UX enhancements, performance optimizations, and feature suggestions that help improve the project.

## Code of Conduct

By participating in this project, you agree to maintain a respectful, constructive, and inclusive environment for everyone.

Be kind, be professional, and focus on helping the project grow in a positive way.

## How to Contribute

You can contribute in several ways:

- Reporting bugs
- Suggesting new features
- Improving documentation
- Fixing issues
- Refactoring or improving code quality
- Enhancing UI/UX behavior
- Improving performance and stability
- Fixing layout, design, or responsiveness issues

## Before You Start

Please take a moment to:

- Read the `README.md`
- Check existing issues and discussions
- Make sure your contribution matches the project goals
- Avoid duplicate pull requests or feature proposals
- Understand the current project structure before making changes

## Development Guidelines

When contributing code, please follow these guidelines:

- Keep the code clean, readable, and modular
- Follow the existing project structure
- Use meaningful names for classes, methods, and variables
- Avoid unnecessary complexity
- Keep changes focused and easy to review
- Prefer small and incremental commits
- Test your changes before submitting them
